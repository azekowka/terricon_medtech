# MedServicePrice.kz — Справочник из данных + умный парсинг

**Дата:** 2026-06-28
**Статус:** утверждён (4 решения зафиксированы пользователем)

## Проблема

Справочник услуг (1230 позиций) грузится из **готового файла**
(`data/source/services_reference.xlsx` → `data/services_dictionary.json`), что
противоречит ТЗ 3.2: *«справочник услуг формируется командой в процессе
разработки на основе анализа собранных данных»*. Источник `seed` генерирует
**выдуманные** цены. Нужно строить справочник ИЗ спарсенных открытых данных и
максимизировать охват реальными данными.

## Цель

- Справочник строится динамически — кластеризацией сырых названий услуг,
  собранных парсингом реальных открытых источников.
- Убрать все зависимости от готовых файлов/синтетики: `services_dictionary.json`,
  `services_reference.xlsx`, `curated_overlay.json`, источники `seed` и `fixtures`.
- Максимум реальных данных со всех источников.
- Умная админка с рабочим, живым, отказоустойчивым запуском парсинга.
- Надёжность для живого показа жюри.

## Источники (всё реальное)

1. **`real`** — реальные прайс-файлы клиник (`data/real_prices/`, PDF/DOCX/XLSX/XLS)
   → лаборатория / диагностика / процедуры.
2. **`idoctor`** (НОВОЕ) — из `data/doctors/doctors.json` извлекаются реальные
   клиники (название, адрес, гео, онлайн-запись) и цены «приём врача» по 18
   регионам. Главный источник охвата (тысячи клиник, вся география).
3. **Живые веб-парсеры** — `kdl, invitro, helix, olymp, medel, mck, aksai, doq`.
   Гибрид: httpx (быстро) + Playwright (JS/Cloudflare). Отказоустойчиво.

## Пайплайн (3 фазы)

1. **Сбор** → `raw_prices` + upsert `clinics` (справочник не нужен).
2. **Построение справочника** из `raw_prices`: нормализация текста → нечёткая
   кластеризация (rapidfuzz + блокинг по токену) → канонические `services`
   (name, synonyms, category, code). + опциональная AI-доразметка (OpenAI).
3. **Нормализация цен** → привязка `service_id`; непривязанное → `unmatched_queue`;
   дедуп + история.

## Компоненты

| Файл | Изменение |

| `frontend/app/admin/page.tsx` | умная панель с живым прогрессом |

## Удаляется из пайплайна

`seed`, `fixtures`, загрузка справочника из файла. (Файлы остаются в репо как
история, но не используются ядром.)

## Нефункциональное

- Данные предсобраны в БД (жюри сразу видит объём); кнопка парсинга —
  асинхронная, с живым прогрессом, отказоустойчивая (упал один источник ≠ стоп).
- Поиск <100 мс сохраняется (индексы).
- Кластеризация использует блокинг → быстро на десятках тысяч названий.

## Модель данных

`Service`: оставляем id/code/name/category/synonyms; добавляем `raw_count` /
`clinic_count` для прозрачности «построено из данных». `base_price/specialty/
tarif` становятся опциональными/неиспользуемыми.
