# MedServicePrice.kz 🩺💸

**Агрегатор и сравнение цен на медицинские услуги в Казахстане** — «Aviasales для медицины».
Один поиск вместо десятков сайтов клиник: введите название анализа, приёма врача или
диагностики — и сразу увидите цены во всех клиниках, отсортированные по выгоде.

> MVP для хакатона (Кейс 1). Полностью рабочий full-stack: парсер → нормализация →
> API → веб-интерфейс с поиском, сравнением, картой, историей цен и подписками.

(≥3 источника, ≥100 услуг, ≥50 нормализованных позиций).

## 🏗️ Архитектура

```
┌────────────┐   raw rows    ┌──────────────┐  normalize   ┌──────────────┐
│  Источники │ ────────────▶ │  raw_prices  │ ───────────▶ │   prices     │
│ файлы/врачи│   (парсеры)   │  (raw-слой)  │ ←построен из │ (нормализ.)  │
│  /живой веб │               │              │   raw-слоя   │              │
└────────────┘               └──────────────┘              └──────┬───────┘
                                    │ unmatched                   │
                                    ▼                             ▼
                            unmatched_queue                 price_history
                            (ручная разметка)               (история цен)

           FastAPI  ──/api/──▶  Next.js (поиск, сравнение, карта, админка)
```

* **Backend** — FastAPI + SQLAlchemy 2.0. Кросс-СУБД: SQLite «из коробки», PostgreSQL в Docker.
* **Парсер** — Python (httpx + BeautifulSoup + **Playwright** для JS/Cloudflare-сайтов),
  вежливый краулинг (задержки, robots.txt), отказоустойчивость (падение одного источника
  не останавливает остальные).
* **Справочник + нормализация** — справочник **строится из собранных сырых названий**
  (кластеризация rapidfuzz по категориям), затем нормализация привязывает цены к нему;
  непривязанное → очередь ручной разметки (+ опциональная AI-доразметка).
* **Frontend** — Next.js 14 (App Router) + TypeScript + Tailwind + Leaflet + Recharts.

Подробнее: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## 🚀 Быстрый старт (локально, без Docker)

Нужны **Python 3.11+**, **Node 18+**, **pnpm**.

### 1. Backend (порт 8000)
```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt

# собрать реальные данные и построить справочник из них (real-файлы + idoctor)
python -m app.cli seed

# (опционально) добавить живые веб-парсеры:
# python -m app.cli seed --live

# запустить API
uvicorn app.main:app --reload --port 8000
```
API: <http://localhost:8000>, Swagger: <http://localhost:8000/docs>.
При первом старте БД наполняется автоматически (если она пустая).

### 2. Frontend (порт 3000)
```bash
cd frontend
pnpm install
# по умолчанию обращается к http://localhost:8000
pnpm dev
```
Откройте <http://localhost:3000>.


### Добавить новый источник (TЗ 4: без переработки ядра)
1. Создайте класс-парсер в `backend/app/parsers/live/<source>.py` (наследник `BaseParser`,
   реализуйте `collect() -> list[RawRecord]`).
2. Зарегистрируйте его в `backend/app/parsers/registry.py`.
Готово — пайплайн нормализации/дедупликации/истории подхватит его автоматически.

## 🧪 Об источниках данных

**Все данные — реальные, никакой синтетики и готовых файлов-справочников** (источники
`seed`/`fixtures` и файл `services_dictionary.json` убраны). Три типа источников проходят
через единый отказоустойчивый пайплайн (`sources → raw-слой → построение справочника →
нормализация`):

1. **`real`** (`parsers/real_extract.py` + `real_prices.py`) — настоящие прайс-листы клиник
   из `data/real_prices/` в форматах **PDF / DOCX / XLSX / XLS**: колоночное распознавание
   для таблиц (столбцы «Наименование» и цена для граждан РК), построчное — для PDF. ~1700 строк.
2. **`idoctor`** (`parsers/doctors_source.py`) — из открытого датасета врачей (idoctor.kz)
   извлекаются **реальные клиники** (адрес, гео, онлайн-запись) и цены **«приём врача»** по
   **18 регионам** Казахстана. Главный источник охвата: ~1280 клиник, ~3800 цен.
3. **Живой веб** (`parsers/live/`) — kdl, invitro, helix, olymp, medel, mck, aksai, doq.
   Гибрид httpx + **Playwright** (JS/Cloudflare), эвристическое извлечение «услуга → цена»,
   вежливый краулинг. Best-effort: недоступный сайт логируется и не останавливает остальные.

**Справочник не загружается из файла** — он формируется кластеризацией собранных сырых
названий (`parsers/dictionary_builder.py`): похожие варианты («ОАК / CBC / Общий анализ
крови») сводятся в одну услугу с синонимами и категорией. Это ровно ТЗ 3.2 «справочник
формируется на основе анализа собранных данных».

Соблюдаются правила ТЗ 8: только открытые данные, задержки между запросами, учёт robots.txt,
никаких персональных данных пациентов.
