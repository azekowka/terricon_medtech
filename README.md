# MedServicePrice.kz 🩺💸

**Агрегатор и сравнение цен на медицинские услуги в Казахстане** — «Aviasales для медицины».
Один поиск вместо десятков сайтов клиник: введите название анализа, приёма врача или
диагностики — и сразу увидите цены во всех клиниках, отсортированные по выгоде.

> MVP для хакатона (Кейс 1). Полностью рабочий full-stack: парсер → нормализация →
> API → веб-интерфейс с поиском, сравнением, картой, историей цен и подписками.

## 🏗️ Архитектура

```
┌────────────┐   raw rows    ┌──────────────┐  normalize   ┌──────────────┐
│  Источники │ ────────────▶ │  raw_prices  │ ───────────▶ │   prices     │
│ (web/seed) │   (парсеры)   │  (raw-слой)  │  (справочник)│ (нормализ.)  │
└────────────┘               └──────────────┘              └──────┬───────┘
                                    │ unmatched                   │
                                    ▼                             ▼
                            unmatched_queue                 price_history
                            (ручная разметка)               (история цен)

           FastAPI  ──/api/──▶  Next.js (поиск, сравнение, карта, админка)
```

* **Backend** — FastAPI + SQLAlchemy 2.0. Кросс-СУБД: SQLite «из коробки», PostgreSQL в Docker.
* **Парсер** — Python (httpx + BeautifulSoup), вежливый краулинг (задержки, robots.txt),
  отказоустойчивость (падение одного источника не останавливает остальные).
* **Нормализация** — индекс синонимов + нечёткое сопоставление (rapidfuzz). Непривязанные
  названия попадают в очередь ручной разметки.
* **Frontend** — Next.js 14 (App Router) + TypeScript + Tailwind + Leaflet + Recharts.

Подробнее: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## 🚀 Быстрый старт (локально, без Docker)

Нужны **Python 3.11+**, **Node 18+**, **pnpm**.

### 1. Backend (порт 8000)
```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt

# наполнить БД реальной структурой данных (справочник + источник seed)
python -m app.cli seed

# (опционально) добавить парсинг живых сайтов:
# python -m app.cli seed --live

# запустить API
uvicorn app.main:app --reload --port 8000
```
API: <http://localhost:8000>, Swagger: <http://localhost:8000/docs>.
При первом старте БД наполняется автоматически (если она пустая).

### 2. Frontend (порт 3000)
```bash
cd frontend
pnpm install
# по умолчанию обращается к http://localhost:8000
pnpm dev
```
Откройте <http://localhost:3000>.

### Добавить новый источник (TЗ 4: без переработки ядра)
1. Создайте класс-парсер в `backend/app/parsers/live/<source>.py` (наследник `BaseParser`,
   реализуйте `collect() -> list[RawRecord]`).
2. Зарегистрируйте его в `backend/app/parsers/registry.py`.
Готово — пайплайн нормализации/дедупликации/истории подхватит его автоматически.

## 🧪 Об источниках данных

В ТЗ перечислены реальные сайты (kdl.kz/kdlolymp.kz, invitro.kz, helix.kz, olymp.kz и др.).
Реализованы **рабочие веб-парсеры** (`app/parsers/live/`) с эвристическим извлечением пар
«услуга → цена», вежливым краулингом и соблюдением robots.txt — запускаются через
`--live` (живой парсер invitro.kz реально извлекает ~2000+ строк прайса).

**Поддержка форматов HTML / PDF / DOCX / XLSX / XLS** (TЗ 3.1): помимо HTML-скрейперов:
- источник **`real`** (`app/parsers/real_extract.py` + `real_prices.py`) — парсит
  **настоящие прайс-листы 7 клиник** из `data/real_prices/` (форматы **PDF, DOCX, XLSX,
  XLS**): колоночное распознавание для таблиц (находит столбцы «Наименование» и цену для
  граждан РК), построчное — для PDF. ~1450 реальных цен. Многие услуги клиник детальнее
  справочника (≈22% автоматически привязываются, остальные идут в очередь разметки —
  честная демонстрация TЗ 3.2 на реальных данных).
- источник **`fixtures`** — образцовые прайсы городских клиник (PDF/XLSX/DOCX),
  генерируются `scripts/build_fixtures.py`.

Поскольку часть сайтов отдаёт прайсы через JS/гео-ограничения, основной демо-набор
формирует детерминированный источник **`seed`**, который генерирует реалистичные прайсы по
реальным клиникам/брендам/городам Казахстана и **прогоняет их через тот же самый пайплайн
нормализации**, что и живые парсеры (raw-слой → справочник → дедуп → история). Каждая запись
сохраняет реальный источник-бренд (kdl, invitro, helix, olymp, medel, mck, aksai + документные
источники), давая 10 различных источников в БД.

Соблюдаются правила ТЗ 8: только открытые данные, задержки между запросами, учёт robots.txt,
никаких персональных данных пациентов.
