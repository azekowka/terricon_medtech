# MedServicePrice.kz - 83 команда (кейс 1)

**Агрегатор и сравнение цен на медицинские услуги в Казахстане** — «Aviasales для медицины».
Один поиск вместо десятков сайтов клиник: введите название анализа, приёма врача или
диагностики — и сразу увидите цены во всех клиниках, отсортированные по выгоде.

> MVP для хакатона (Кейс 1). Полностью рабочий full-stack: **парсер → нормализация к
> собственному справочнику → API → адаптивный веб-интерфейс** с поиском, сравнением,
> картой, историей цен, подписками, а также каталогом врачей, заболеваний и AI-ассистентом.
>
> 📐 Подробная архитектура и обоснование решений — в **[SYSTEM_DESIGN.md](SYSTEM_DESIGN.md)**.

## 🏗️ Архитектура

```
┌──────────────────────────────────────┐
│  ИСТОЧНИКИ (плагины, контракт RawRecord)│
│  • seed     — из СВОЕГО справочника услуг │   list[RawRecord]
│  • fixtures — образцовые прайсы (PDF/XLSX/DOCX) ├───────────┐
│  • live     — веб-скрейперы (по --live, опц.)  │           ▼
└──────────────────────────────────────┘     ┌───────────────┐   ┌──────────────┐   ┌──────────────┐
                                              │  pipeline.py   │──▶│  raw_prices   │──▶│   prices      │
                                              │ run_source()   │   │  (сырой слой) │   │ (нормализ.,   │
                                              └──────┬────────┘   └──────────────┘   │  дедуп, UI)   │
                                                     │ нет совпадения                 └──────┬───────┘
                                                     ▼                                        ▼
                                            unmatched_queue                            price_history
                                            (ручная разметка)                          (история цен)

              FastAPI  ──/api/──▶  Next.js 14 (поиск, сравнение, карта, врачи, болезни, ассистент, админка)
```

* **Backend** — FastAPI + SQLAlchemy 2.0. **Кросс-СУБД:** SQLite «из коробки», PostgreSQL в
  Docker (переключение одной `DATABASE_URL`).
* **Парсер** — Python (pdfplumber / python-docx / openpyxl / xlrd / BeautifulSoup), вежливый
  краулинг (задержки, robots.txt), отказоустойчивость (падение одного источника/файла/строки
  не останавливает остальное).
* **Нормализация** — индекс синонимов + нечёткое сопоставление (rapidfuzz WRatio, порог 86).
  Непривязанные имена попадают в очередь ручной разметки с подсказкой.
* **Frontend** — Next.js 14 (App Router) + TypeScript + Tailwind + **MapLibre GL** + Recharts.

Полное описание — **[SYSTEM_DESIGN.md](SYSTEM_DESIGN.md)**.

## 🚀 Быстрый старт (локально, без Docker)

Нужны **Python 3.11+**, **Node 18+**, **pnpm**.

### 1. Backend (порт 8000)
```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt

# наполнить БД: справочник + данные из своего справочника (seed) + образцовые прайсы (fixtures)
python -m app.cli seed

# запустить API
uvicorn app.main:app --reload --port 8000
```
API: <http://localhost:8000>, Swagger: <http://localhost:8000/docs>.
При первом старте БД наполняется автоматически (если пустая) — включая врачей и заболевания.

### 2. Frontend (порт 3000)
```bash
cd frontend
pnpm install
# по умолчанию обращается к http://localhost:8000
pnpm dev
```
Откройте <http://localhost:3000>.

### Добавить новый источник (ТЗ 4: без переработки ядра)
1. Создайте класс-парсер (наследник `BaseParser`, метод `collect() -> list[RawRecord]`).
2. Добавьте одну строку в `backend/app/parsers/registry.py` (`PARSERS`).
Готово — пайплайн нормализации / дедупликации / истории подхватит его автоматически.

## 🧪 Об источниках данных

Набор данных строится из **собственного справочника услуг** — не парсятся чужие конкретные
прайс-файлы:

- **Справочник** (`data/services_dictionary.json`, 1230 услуг) строится скриптом
  `scripts/build_dictionary.py` из **официального справочника ТЗ** `services_reference.xlsx`:
  категория выводится из префикса тариф-кода, добавляются синонимы и курируемый overlay.
  Это наш единый нормализованный справочник (ТЗ 3.2; ≥50 — выполнено с запасом).
- Источник **`seed`** — детерминированный генератор: создаёт реалистичные прайсы по реальным
  брендам/городам Казахстана **из этого же справочника** и **прогоняет их через тот же
  пайплайн нормализации** (raw-слой → справочник → дедуп → история), эмитя в т.ч. синонимы и
  заведомо-неизвестные имена, чтобы честно нагрузить нормализацию и очередь разметки (ТЗ 3.2).
- Источник **`fixtures`** — образцовые публичные прайсы городских клиник в **3 форматах**
  (PDF/XLSX/DOCX), генерируются `scripts/build_fixtures.py` — сквозная проверка мультиформатного
  ингеста (ТЗ 3.1).
- Источники **`live`** (`kdl` / `invitro` / `helix`) — рабочие веб-скрейперы с эвристическим
  извлечением «услуга → цена», вежливым краулингом и соблюдением robots.txt. Запускаются
  **опционально** через `--live`; это best-effort путь расширения (часть сайтов отдаёт прайсы
  через JS / гео-ограничения / Cloudflare), поэтому в основной набор данных они не входят.

Каталоги **врачей (40 467)** и **заболеваний (274)** — отдельные датасеты, спарсенные с
idoctor.kz (`scripts/scrape_idoctor_api.py`, `scripts/scrape_lechenie.py`).

Соблюдаются правила ТЗ 8: только открытые данные, задержки между запросами, учёт robots.txt,
никаких персональных данных пациентов.
