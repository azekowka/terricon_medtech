# Changelog — MedServicePrice.kz

Все значимые изменения проекта.

## [Unreleased]

### Changed
- **Шрифт — как у Aviasales**: подключён Inter через `next/font` (тот же шрифт, что
  использует Aviasales; их фирменный Stapel проприетарный → fallback на Inter).
  Глобальный стек шрифтов приведён к точному стеку Aviasales:
  `Stapel, -apple-system, BlinkMacSystemFont, Inter, Roboto, Helvetica, Arial, sans-serif`

- **Селектор города на странице болезни** заменён нативного `<select>` на
  shadcn-style комбобокс (`components/CitySelect.tsx`): кнопка-триггер (MapPin +
  ChevronsUpDown), всплывающая панель с галочками (Check), счётчиками врачей,
  поиском (при >8 пунктов), закрытием по клику вне/Escape. Переиспользуемый компонент.

### Fixed
- **Гонка запросов при смене города на странице болезни**: дефолтный запрос (Алматы)
  мог перезаписать ответ для сохранённого города (Шымкент показывал 236 вместо 23).
  Добавлен guard отмены (`active`) в effect — всегда побеждает последний выбранный город.

## [0.9.0] — Раздел «Лечение заболеваний»

### Added
- На странице болезни — **выбор города** (дропдаун с «Все города» + 18 регионов):
  показывает врачей, которые лечат болезнь, по выбранному городу; выбор сохраняется
  (общий с разделом «Врачи»). Рефетч без перезагрузки всей страницы.
- Спарсен каталог болезней с idoctor (`ext-api/v2/illnesses/{alias}`) обходом графа
  `similar` от разнообразных seed-болезней (из профилей врачей): **274 заболевания,
  169 категорий** (по специальностям). Хранятся только факты (название, лечащие
  специальности, связи), без прозаических описаний. Скрейпер: `scripts/scrape_lechenie.py`.
- **Backend**: таблица `illnesses`, загрузчик, API `routers/illnesses.py`:
  `/api/illnesses/categories` (каталог по специальностям), `/api/illnesses` (поиск),
  `/api/illnesses/{alias}` (болезнь + лечащие специальности + **врачи, которые её лечат**
  — наши врачи по специальности и городу + похожие болезни).
- **Frontend**: `/lechenie` — каталог (картинка 1: карточки категорий → модалка с
  болезнями, картинка 2) + поиск; `/lechenie/[alias]` — страница болезни (как у idoctor):
  название, лечащие специальности, врачи (карточки, фильтр по городу), похожие болезни.
  Пункт «Лечение» в навигации.

### Fixed
- **Чат-ассистент не вёл по городам/клиникам**: запрос «клиники в Актобе» упирался в
  тупик. Добавлены интенты «клиники» и «город»: бот подсказывает по городу (кол-во
  врачей и клиник, цена приёма от) + кнопки (врачи города, карта цен). Город-уточнение
  тоже наследуется в мульти-тёрне. Сообщение об ошибке сети в виджете заменено с
  «Ничего не найдено» на понятное «Не удалось получить ответ…».
- **Фото врачей не загружались**: аватары из API idoctor приходят относительными путями
  (`/images/...`), браузер искал их на нашем домене → битые картинки. Добавлены:
  хелпер `doctorAvatar()` (абсолютизация URL + распознавание плейсхолдера `no-userpic`),
  компонент `DoctorAvatar` с фолбэком-иконкой при ошибке загрузки. Скрейпер тоже
  сохраняет абсолютные URL. Применено в карточках, профиле и рекомендациях.

## [0.8.0] — Чат-ассистент по услугам

### Added (OpenAI)
- **Интеграция OpenAI** (`backend/app/llm.py`): ответы ассистента генерирует
  `gpt-4o-mini`, **заземлённый на наши данные** (FACTS из движка) и доменно-
  ограниченный системным промптом (только услуги платформы; off-topic вежливо
  отклоняется; цены/числа — только из FACTS, без выдумок). Ключ в `backend/.env`
  (gitignore). **Фолбэк** на детерминированный движок, если ключа нет/ошибка.
- **Мульти-тёрн**: история диалога передаётся в модель; короткие уточнения
  («а в Караганде?») наследуют контекст (специализацию/услугу) и заземляются на
  реальные данные нужного города; ссылки-действия всегда из нашей БД.

### Added (engine)
- **Бэкенд-ассистент** (`backend/app/routers/assistant.py`, `POST /api/assistant/chat`):
  доменно-ограниченный помощник, отвечает ТОЛЬКО по услугам платформы — цены анализов,
  поиск врачей, сравнение клиник, самый дешёвый город, помощь по сайту. Определяет
  намерение и сущности (услуга — точное совпадение синонима/аббревиатуры + fuzzy;
  специализация; город) по живой БД и возвращает ответ + кнопки-действия (ссылки).
  Внешний LLM не требуется; off-topic вежливо отклоняется. Локали RU/KK/EN.
- **Чат-виджет** (`components/ChatWidget.tsx`) в стиле prompt-kit (собственная
  реализация): плавающая кнопка, панель с пузырями сообщений, чипы-подсказки,
  индикатор набора, авто-textarea, кнопки-действия. Виден на всех страницах,
  локализован.

## [0.7.0] — Карта в стиле Aviasales (Mapbox GL)

### Changed
- **Leaflet заменён на Mapbox GL** (движок MapLibre GL — open-source Mapbox GL,
  работает без токена; для настоящих Mapbox-тайлов задать `NEXT_PUBLIC_MAPBOX_TOKEN`).
- Страница `/map` — **комбинированная карта в стиле Aviasales**: маркеры-«пузыри»
  цен по городам (город + «от X ₸», самый дешёвый подсвечен) **+ маркеры клиник**
  (синие точки с попапом: название, адрес, часы, кол-во услуг, рейтинг, ссылка) **+
  выпадающий выбор города** (как раньше): зум к городу и фильтр его клиник. Базовая
  карта — CARTO Voyager. Мини-карта клиники тоже переведена на MapLibre.

### Added
- Эндпоинт `GET /api/map/cities` — минимальная цена приёма по каждому городу.

## [0.6.0] — Локализация (RU/KK/EN) + рекомендательная система

### Added
- **i18n** (`lib/i18n/`): словарь RU/Қазақша/English, `I18nProvider` с
  автоопределением языка (по браузеру), переключателем в шапке и сохранением выбора в
  `localStorage`. Применяется ко всем страницам, меняется в любой момент. Локализованы
  шапка/футер, главная, весь раздел «Врачи» (список, фильтры, карточки, попап городов,
  профиль), строка поиска; локале-зависимые формы единиц (год/жыл/year, отзыв/пікір/review).
- **Рекомендательная система** (в стиле Aviasales/Trip.com): эндпоинт
  `GET /api/doctors/recommendations` подбирает несколько действительно выгодных вариантов
  с вычисленными причинами — «💎 Лучшая цена/качество» (на N% ниже средней цены),
  «📉 Самый выгодный» (дешевле средней на X₸), «🏆 Топ по рейтингу» (выше рейтинг).
  Компонент `DoctorRecommendations` показывает их строкой на странице врачей.

## [0.5.0] — Страницы врачей (профиль как на idoctor)

### Added
- Найден API детального профиля: `ext-api/v2/doctors/{alias}` (биография/образование
  `description`, заболевания `illnesses`, клиники с графиком), `…/services` (услуги),
  `comments?type=doctor&id=&mood=positive|negative` (отзывы).
- **`app/doctors_profile.py`** — fetch+cache профиля по запросу (on-demand) с
  кэшированием в БД (колонки `description`, `services`, `review_items`, `has_comments`,
  `online_bookings`, `profile_fetched`). Эндпоинт `GET /api/doctors/{id}/profile`.
- **Страница врача `/doctors/[id]`** — клон профиля idoctor: шапка (фото, имя, бейджи,
  специализации, стаж/категория, рейтинг, кол-во отзывов), «О враче» (биография/
  образование), «Лечение заболеваний», «Услуги», «Где принимает» (клиники с графиком
  по дням, адрес, карта, цена), «Отзывы» (автор, рейтинг, теги, дата, ответ клиники),
  липкий блок записи. Карточки врачей теперь ведут на нашу страницу, а не на idoctor.

## [0.4.0] — Каталог врачей (клон вкладки idoctor /doctors)

### Added
- **Скрейпер idoctor.kz**: нашёл публичный JSON API
  (`https://ext.idoctor.kz/ext-api/v2/doctors?page=N&perPage=100` + заголовок
  `x-user-city: <регион>`) с полной пагинацией. `scripts/scrape_idoctor_api.py`
  собирает **полный каталог — 40 467 врачей по всем 18 регионам Казахстана**
  (Алматы 7439, Астана 3503, Семей 3375, Караганда 3218, Туркестан 3078 …) с
  полными данными: ФИО, специализации, категория, стаж, рейтинг, отзывы, клиники с
  **графиком работы по дням**, адресом, гео-координатами и ценами приёма; плюс
  «Лечение заболеваний» с профиля. Устойчив к Cloudflare (backoff + обновление
  cookie), атомарная запись + resume для докачки по батчам.
  (`scripts/scrape_idoctor.py` + `idoctor_extract.py` — резервный путь через SSR
  `initialDoctorsList`.)
- **Backend**: таблица `doctors`, загрузчик (`doctors_loader.py`), API
  (`routers/doctors.py`): `/api/doctors/meta` (регионы+специализации), `/api/doctors`
  (фильтры: регион, специализация, цена, рейтинг, стаж, дети, онлайн-запись,
  сортировка, пагинация), `/api/doctors/{id}`.
- **Frontend `/doctors`** — клон каталога врачей: попап выбора города (18 регионов),
  сайдбар фильтров (специализация с поиском+счётчиками, цена, рейтинг, стаж,
  переключатели), карточки врачей (фото, специализации, стаж/категория, приём
  детей, рейтинг+отзывы, клиника+адрес, цена со скидкой, онлайн-запись, «Записаться»),
  пагинация, модалка врача (клиники, график, заболевания, запись).
- Пункт «Врачи» в навигации.

## [0.2.0] — Официальный справочник услуг

### Changed
- Словарь услуг заменён на **официальный справочник из ТЗ**
  (`data/source/services_reference.xlsx`) → **1230 услуг, 122 специальности**.
  Категория выводится из префикса тарифного кода (A/B/C/D), курируемые синонимы/цены

### Fixed
- `/api/services` обрезал список до 500 → на главной «Процедура» = 0 услуг.
  Добавлены счётчики по категориям в `/api/meta`, сортировка `?sort=popular`.
